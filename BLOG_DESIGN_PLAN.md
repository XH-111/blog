# 个人博客系统详细设计方案

## 1. 项目定位

本项目定位为一个个人博客与轻量 CMS 系统，核心目标是服务个人技术表达、作品展示、留言互动和后台内容管理。

网站不以资讯流为主，而是以“个人主页 + 精选内容 + 后台可配置”为主。首页承担入口和品牌展示，文章页承担阅读分发，后台承担内容生产、媒体管理、AI 写作辅助和站点配置。

## 2. 用户角色

### 访客

- 浏览首页、精选文章、分类文章和文章详情。
- 查看关于页、项目作品和个人经历。
- 在留言板留言。
- 在文章下发表评论、点赞。

### 站长 / 管理员

- 管理文章、分类、标签、评论、留言和媒体库。
- 配置首页封面、首页文案、关于页资料。
- 使用 AI 摘要、AI 评论、AI 润色辅助写作。
- 审核评论和留言。

## 3. 前台页面设计

### 首页

首页作为站点入口，不再堆叠大量文章，而是突出个人博客品牌和核心入口。

当前支持：

- 首页标题、副标题、介绍文案可配置。
- 标题、副标题、介绍文案可为空；为空时前台隐藏。
- 标题、副标题、介绍文案颜色可配置。
- 首页封面支持图片和视频。
- 图片 / 视频封面支持位置调整和缩放。
- 主要按钮跳转文章页。
- 次要按钮跳转关于页。
- 下方保留三个入口卡片：精选文章、项目作品、关于我。

建议后续增强：

- 支持首页入口卡片后台配置。
- 支持首页背景遮罩透明度配置。
- 支持多套首页主题预设。
- 支持移动端单独封面。

### 文章页

文章页承担阅读入口功能，而不是传统归档页。

当前设计：

- 默认展示精选文章。
- 支持“全部文章”和分类切换。
- 每页最多 10 篇。
- 文章卡片为短卡片结构，鼠标悬停有突出效果。
- 分类、精选、全部文章都走同一套分页逻辑。

建议后续增强：

- 搜索框固定在文章页顶部。
- 支持按热度、时间排序。
- 支持文章系列。
- 支持阅读进度和目录。

### 文章详情页

当前文章详情支持：

- Markdown 内容展示。
- 表格渲染。
- 文章封面。
- 分类、标签、阅读时间、发布时间等基础信息。
- 点赞和评论。

建议后续增强：

- 文章目录锚点。
- 代码高亮。
- 图片点击预览。
- 上一篇 / 下一篇。
- 私密文章密码访问闭环。

### 关于页

关于页用于展示个人资料、技术栈、项目作品、时间线和联系方式。

当前支持：

- 后台配置标题、身份标签、副标题、头像、简介、联系方式。
- 时间线结构化配置。
- 项目作品配置。
- 微信二维码和 GitHub。
- 隐藏站点数据和写作主题。

建议后续增强：

- 项目作品单独页面。
- 简历 PDF 下载入口。
- 时间线支持拖拽排序。

### 留言板

留言板用于访客交流。

当前支持：

- 访客留言。
- 站长回复。
- 后台审核。
- 站长头像使用关于页照片。

建议后续增强：

- 防刷限流。
- 验证码。
- 敏感词过滤。
- 邮件通知。

## 4. 后台模块设计

### 仪表盘

用于展示站点基本数据、待处理评论、待处理留言和最近文章。

建议后续增强：

- 最近 7 天 PV / UV。
- 热门文章。
- 媒体空间占用。
- AI 使用次数。

### 文章管理

当前支持：

- 文章列表。
- 新建、编辑、发布、草稿、回收站。
- 精选文章配置。
- 批量精选。
- 文章封面上传和媒体库选择。
- 正文图片上传和媒体库插入。

建议后续增强：

- 批量删除。
- 批量修改分类 / 标签。
- 文章复制。
- 自动保存版本历史。

### AI 写作助手

当前支持：

- AI 摘要。
- AI 评论。
- AI 润色。
- 润色支持选中区域。
- 润色支持撤销。
- AI 评论支持评论重点和自定义要求。
- 可选千问联网搜索。
- 润色结果支持说明框和放大查看。

建议后续增强：

- AI 任务历史。
- 多模型配置。
- 用户自定义提示词模板。
- AI 输出差异对比。

### 媒体库

当前支持：

- 图片上传。
- 视频上传。
- 多文件上传。
- 媒体记录入库。
- 被文章封面或正文引用时禁止删除。

建议后续增强：

- 视频缩略图。
- 文件大小展示。
- 按类型筛选：图片 / 视频。
- 上传进度条。
- 阿里云 OSS 存储。

### 首页配置

当前支持：

- 首页标题、副标题、介绍文案。
- 文案可为空。
- 文案颜色配置。
- 图片封面。
- 视频封面。
- 封面拖拽位置。
- 封面缩放。
- 保存前实时预览。

建议后续增强：

- 首页卡片配置。
- 首页按钮颜色配置。
- 封面遮罩配置。
- PC / 移动端独立配置。

### 关于页配置

当前支持：

- 基础资料。
- 联系方式。
- 头像。
- 微信二维码。
- 时间线。
- 项目作品。
- 合作入口。

建议后续增强：

- 时间线排序。
- 项目作品排序。
- 项目作品独立封面裁剪。

## 5. 技术架构

### 前端

- React
- Vite
- TypeScript
- Hash 路由
- CSS 原生样式

前端构建产物为 `dist/`，生产环境建议由 Nginx 托管。

### 后端

- Node.js 原生 HTTP server
- PostgreSQL
- 文件上传保存到 `public/uploads/yyyy/mm/dd`
- 后台接口通过 Bearer token 鉴权

生产环境建议：

- Node 后端用 PM2 或 systemd 守护。
- 后端只监听内网地址或本机端口，由 Nginx 反代。
- `/api` 反代到后端。
- `/uploads` 可由后端服务，也可由 Nginx 直接映射静态目录。

### 数据库

- PostgreSQL
- 当前本地使用 Docker 容器 `blog-postgres`
- 生产环境建议继续使用 Docker 跑 PostgreSQL，并挂载 volume。

核心表：

- `admin_users`
- `posts`
- `categories`
- `tags`
- `post_tags`
- `comments`
- `messages`
- `media_assets`
- `site_settings`
- `ai_tasks`

## 6. 部署设计

推荐阿里云服务器部署结构：

```text
Nginx
├─ /          前端 dist 静态文件
├─ /api       反代 Node 后端 127.0.0.1:8000
└─ /uploads   静态映射或反代上传目录

Node Backend
└─ 连接 PostgreSQL

PostgreSQL Docker
└─ 挂载数据卷到服务器固定目录
```

### 服务器建议

当前阿里云配置：

- Ubuntu 22.04
- 2 vCPU
- 2 GiB 内存
- 40 GiB ESSD

可以满足个人博客使用，但要注意：

- 不要把 PostgreSQL 端口暴露公网。
- 视频文件不要过大。
- Nginx 上传大小限制要调整。
- 定期备份数据库和上传目录。

### 生产环境变量

建议配置：

```env
PORT=8000
HOST=127.0.0.1
DATABASE_URL=postgres://blog:强密码@127.0.0.1:5432/blog_prod
CORS_ORIGIN=https://你的域名
ADMIN_DEFAULT_PASSWORD=强后台密码
DASHSCOPE_API_KEY=你的千问 Key
QWEN_MODEL=qwen-plus
QWEN_RESPONSES_MODEL=qwen-plus
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
QWEN_TIMEOUT_MS=30000
QWEN_RESPONSES_TIMEOUT_MS=90000
AI_WEB_SEARCH_ENABLED=true
```

### Nginx 关键配置点

- 配置 HTTPS。
- `/api` 反代到 `http://127.0.0.1:8000`。
- `/uploads` 支持访问上传文件。
- `client_max_body_size` 至少设置为 `50m`。
- 前端使用 history/hash 路由时，根路径回退到 `index.html`。

## 7. 安全设计

上线前必须完成：

- 修改默认后台密码。
- 生产数据库密码不能使用 `blog123456`。
- 不提交 `.env`。
- 阿里云安全组只开放 `22`、`80`、`443`。
- 不开放 `5432` 和 `8000` 到公网。
- 开启 HTTPS。

建议增强：

- 后台登录失败限流。
- 评论和留言限流。
- 文件上传类型白名单。
- 文件上传大小限制。
- AI 接口调用频率限制。
- 管理端操作日志。

## 8. 备份设计

需要备份两类数据：

### 数据库

建议每天执行：

```bash
pg_dump "$DATABASE_URL" > backup-$(date +%F).sql
```

### 上传文件

备份目录：

```text
public/uploads/
```

建议：

- 每天打包一次。
- 保留最近 7 天。
- 重要阶段手动备份。

## 9. 当前上线前风险

### 高优先级

- 后端监听地址仍需支持 `HOST` 配置，避免服务器部署访问问题。
- 生产环境部署文件还未完善。
- 上传目录持久化方案未落文档。
- 数据库备份方案未自动化。
- 默认密码和数据库密码需要替换。

### 中优先级

- 后端部分错误文案存在乱码。
- 媒体库视频缺少缩略图和时长信息。
- 评论、留言、AI 接口缺少限流。
- 后台 token 存 localStorage，必须依赖 HTTPS。

### 低优先级

- 后台部分模块还可以继续细化体验。
- README 和 backend README 已落后于当前功能。
- 缺少完整部署文档。

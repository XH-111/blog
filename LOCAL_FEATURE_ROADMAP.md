# 本地功能体验补齐路线图

本文档是当前 active goal 的详细执行蓝图。目标是在不处理部署事项、不做 AI 调用限流的前提下，补齐个人博客当前最影响体验、最没有闭环的本地功能。

## 总原则

- 不处理部署、服务器、Docker、Nginx、HTTPS、域名、安全组。
- 不做 AI 调用限流。
- 不重复实现已经完成的功能。
- 优先补齐已有功能闭环，而不是引入大功能。
- 每个阶段只做该阶段范围内的改动。
- 每个阶段完成后运行必要验证。
- 每个阶段验收成功后必须单独提交一次本地 git commit。
- 不提交 `.env`、日志、`dist/`、`node_modules/`、`public/uploads/`。
- 不提交真实密钥、真实数据库密码、真实 API Key。
- 如果某阶段发现已有功能已经完成，只补充文档说明并跳过代码改动，不强行重做。
- 如果某阶段无法在本地合理完成，记录原因，继续下一个阶段。

## 阶段 1：首页入口卡片后台配置

### 任务

- 将首页下方入口卡片从前端写死改为后台首页配置的一部分。
- 每张卡片支持：标题、描述、按钮文案、跳转链接、图标类型、是否显示。
- 默认保留“精选文章 / 项目作品 / 关于我”三张卡作为兜底。
- 前台首页根据后台配置渲染入口卡片。

### 不做

- 不做复杂图标上传。
- 不做拖拽排序，最多使用上移/下移按钮或数组顺序。

### 验收目标

- 后台 `/admin/home-config` 能编辑入口卡片。
- 保存后刷新前台首页，卡片内容变化。
- 隐藏某张卡片后，前台不显示。
- 如果后台配置为空，前台仍显示默认三张卡。

### 验证命令

```powershell
npm.cmd run build
node --check backend\src\server.js
```

### 停止条件

上述验收全部满足后停止本阶段。

### 提交信息

```text
feat: make home entry cards configurable
```

## 阶段 7：私密文章和密码文章访问闭环

### 任务

- 检查当前 `visibility=private/password` 的真实行为。
- 私密文章应只在后台可见，前台不可访问。
- 密码文章如果暂不实现密码输入，就不要在前台误展示为可访问。
- 后台编辑器给出清晰说明。

### 不做

- 如果复杂度过高，可以先不实现完整密码输入页，但必须避免“看起来支持但实际不支持”的半成品体验。

### 验收目标

- `private` 文章前台列表和详情不可见。
- `password` 文章行为明确：要么有密码输入闭环，要么前台不公开展示并给后台提示。
- 后台编辑器对三种可见性有说明。
- 不影响公开文章访问。

### 验证命令

```powershell
npm.cmd run build
node --check backend\src\server.js
```

### 停止条件

可见性行为不再误导用户后停止。

### 提交信息

```text
fix: clarify post visibility behavior
```
